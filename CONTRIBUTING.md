# Contributing to Learn Rust

Thank you for your interest in contributing to this project! We welcome contributions from everyone.

## How To Contribute

1. Fork this repository and clone it to your local machine.
2. Create a branch from the `develop` branch.
3. Make sure your changes are focused and easy to review.
4. Run `cargo fmt` to ensure that your code is properly formatted.
5. Run `cargo clippy` to ensure there are no linting errors.
6. Add or update tests when making code changes.
7. Ensure that all tests pass locally (`cargo test`).
8. Open a pull request to merge your changes into the `develop` branch.

## Branch Naming Conventions

When creating branches, use one of the following naming conventions:

- `feature/<short-description>`
- `bugfix/<short-description>`
- `docs/<short-description>`
- `chore/<short-description>`

## Code Style

- **Use `snake_case` for variable names**.
- **Use `CamelCase` for structs, enums, and type names**.
- **Include comments where necessary** to explain your code.

## Pull Request Expectations

- Clear title and description of what your PR does.
- Link to relevant issues, if applicable.
- Ensure your PR is focused on one logical change.
- Provide details on any tests you added or updated.

We appreciate your contributions and thank you for helping make this project better!
